# 變數宣告的想像

可以把變數宣告想像成「準備一個有規則的容器盒子，然後貼上識別用的標籤名稱，再把資料放進去」。

## 對應概念

- 變數名稱：像是容器盒子外面貼的標籤名稱，用來識別這個盒子
- 宣告型別：像是容器盒子的屬性，決定這個盒子可以裝什麼資料
- 指派的值：就是目前放進盒子裡的內容

## 例子說明

```java
int x1 = 1;
```

## 圖示

```text

拆開來看:

int      x1      =      1
|        |              |
|        |              +--> 放進去的值
|        +-----------------> 盒子名稱 / 標籤
+--------------------------> 盒子型別 / 只能裝整數
```

```text
盒子
    _____________
   /     1      /|
  /____________/ |
  |            | |
  |   名稱：x1  | |
  |   (int)    | /
  |____________|/
```

## 口語化理解

「我準備了一個只能裝整數的盒子，這個盒子的名字叫做 `x1`，目前裡面放的是 `1`。」

## 使用變數的時機

當程式中會大量使用相同數值時，就可以使用變數。

這樣做的好處是：

- 可以讓程式比較好讀
- 可以避免一直重複寫相同的數值
- 如果之後要修改數值，只需要改變數，不需要每一行都改

## 範例

### 不用變數

```java
System.out.println(5 + 3);
System.out.println(5 * 3);
System.out.println(5 / 3);
System.out.println(5 - 3);
```

上面這種寫法是直接把數字寫死在程式裡。

如果之後想把 `5` 或 `3` 改掉，就要一行一行修改。

### 使用變數

```java
int x = 2;
int z = 3;
System.out.println(x + z);
System.out.println(x * z);
System.out.println(x / z);
System.out.println(x - z);
```

這種寫法就是先把數值放進變數裡，再拿變數去做運算。

可以想成：

- `x` 是一個整數盒子，裡面裝 `2`
- `z` 是一個整數盒子，裡面裝 `3`
- 後面的運算都不直接寫數字，而是使用這兩個盒子裡的值

## 圖示

```text
x 這個盒子
    _____________
   /     2      /|
  /____________/ |
  |            | |
  |   名稱：x   | |
  |   (int)    | /
  |____________|/

z 這個盒子
    _____________
   /     3      /|
  /____________/ |
  |            | |
  |   名稱：z   | |
  |   (int)    | /
  |____________|/
```

## 總結

當同一個數值會重複使用很多次時，使用變數會比直接寫死數字更方便，也更容易維護。
